## サービス概要

ゲームを一緒に遊ぶ仲間の「この日に集まれる！」を共有できるサービスです。
ユーザーは自分の予定を登録し、フォローしている仲間の予定をひと目で確認できます。
ゲーム前に予定調整で悩む手間を減らすことを目的にしています。

## このサービスへの思い・作りたい理由

ゲーム仲間と遊ぶ時、いつも最初に行うのが「何日の何時に集まれるか」という予定調整でした。
毎回メッセージや通話で確認し合う必要があり、人数が増えるほど把握が難しくなることが多かった経験があります。
この煩わしさを減らして、遊ぶまでの準備をもっと簡単にしたいという思いが、このサービスを作る理由になっています。

## ユーザー層について

主なユーザーは、普段からフレンドと一緒にゲームをする人たちです。
ゲームを一人ではなく、誰かと合わせて遊ぶ人にとって、予定の共有は必ず必要になるからです。
また、ゲーム上の「フレンドリスト」でつながっているだけでは予定が見えないため、この層は特に利便性が高くなると考えています。

## サービスの利用イメージ

ユーザーはログインすると、まずフォローしている仲間の「今週の予定一覧」をすぐに確認できます。
自分の予定を登録すると、仲間にも自分が遊べる時間帯がわかるようになります。
予定確認のためにチャットを何往復もしなくて済むので、遊ぶまでの流れが短くなり、素早く集まれる状態を作れます。
この「すぐに予定が把握できる状態」がユーザーにとっての価値になります。

## ユーザーの獲得について

ゲーム仲間同士の紹介を中心に広がる形を想定しています。
サービスの性質上、単独で使うより、仲間と一緒に使うことで価値が生まれるため、自然に共有されやすいと考えています。
また、ユーザー一覧からフォローできる仕組みにすることで、最初の利用者が次の利用者を呼ぶ構造も作れます。

## サービスの差別化ポイント・推しポイント

一般的なカレンダーアプリは汎用的で、細かい時間設定や複雑な表示が中心になっています。
このサービスは「ゲーム仲間の予定だけ」「今週だけ」「昼・夕方・夜の区分だけ」という必要最小限に絞った設計にしている点が差別化の部分です。
予定調整に必要な情報だけに限定することで、操作が早く、画面もわかりやすく、使う側の負担が軽くなります。
この軽さがこのサービスの推しポイントになります。

## 機能候補

最初に実装したい機能は、ユーザー登録、ログイン、自分の予定登録、フォロー機能、フォローしているユーザーの今週の予定一覧の表示です。
本リリースまでに追加したい機能としては、予定のメモ入力、プロフィール編集、予定の削除や編集の細かな改善などを想定しています。
どれもサービスの中心となる「予定把握」を邪魔しない範囲で少しずつ広げていく形を考えています。

## 使用する技術スタック

アプリケーションは Rails を使用して作成する予定で、バージョンは安定して利用されているものを選びます。
データベースは MySQL を利用し、予定・ユーザー・フォロー関係を保存します。
デプロイ先は Render または Rails で一般的に利用されるサービスを想定し、継続的に動かせる環境に置きます。
MVPではRailsのビューで一覧・スケジュール画面を実装し、本リリースではこれらの画面を中心にReactを導入することで、視覚的に分かりやすく、操作感の良いUIへの改善を予定しています。
