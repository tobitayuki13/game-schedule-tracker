## サービス概要
ユーザーが、「やらなければいけないが、めんどうで始められない行動」を、最初の一手から実行しやすくするためのルーティン管理アプリです。
ユーザーは、ユーザ自身があらかじめ決めた順番通りにタスクを実行し、開始と終了の操作だけで行動を進められます。
考える負担を減らし、「始める前で止まる」状態を防ぐことを目的としています。

## このサービスへの思い・作りたい理由
日常生活の中で、掃除や洗濯などやるべきことが分かっていても、「めんどう」という理由で後回しにしてしまうことが多くありました。
タスク管理アプリを使っても、タスクを確認した時点で気が重くなり、結局別の娯楽を優先してしまう経験を何度もしています。

この経験から、行動できない原因は「タスクの量」や「内容」ではなく、「始める前に考えることが多いこと」にあると感じました。
そこで、行動の選択や判断を極力減らし、最初の一手だけに集中できる仕組みを作りたいと考え、本サービスを企画しました。

## ユーザー層について
主なターゲットは、日常的な家事や作業をつい先延ばしにしてしまう人です。
特に、タスク管理アプリを使った経験はあるものの、実際の行動につながらなかった人を想定しています。

この層は「やる気がない」のではなく、「やる気はあるけれど、始めるハードルが高い」ことが問題である場合が多いため、本サービスの仕組みが有効だと考えています。

## サービスの利用イメージ
ユーザーは最初に、毎日実行したい行動を順番固定のルーティンとして設定します。
利用時はトップ画面から「開始ボタン」を押すだけで、最初のタスクに取りかかります。
タスク完了後に「終了ボタン」を押すことで次の行動へ進み、ルーティンを順番に消化していきます。

これにより、何をするか悩む時間が減り、自然に行動を開始できる体験を提供します。

## ユーザーの獲得について
タスク管理や習慣化に悩みを持つ人が多く利用するSNSや記事媒体を通じて認知を広げることを想定しています。
特に「先延ばし」「めんどう」「やる気が出ない」といった悩みに共感する層に向けて、体験ベースで価値を伝える形を考えています。

現時点では大規模な集客は想定せず、個人利用や小規模なユーザーからの利用を想定しています。

## サービスの差別化ポイント・推しポイント
多くのタスク管理アプリは、自由度が高い反面、ユーザーに判断を委ねる場面が多く存在します。
本サービスでは、タスクの「順番を固定」し、「開始と終了のみ操作する」という点に特徴があります。

これにより、ユーザーは「何からやるか」「今やるべきか」といった判断を行う必要がなくなり、行動開始までの心理的負担を大きく減らせます。
この「考えさせない設計」が本サービスの最大の推しポイントです。

## 機能候補
MVPリリース時に作成したい機能

ユーザー登録・ログイン機能

ルーティン作成機能（順番固定）

開始ボタン・終了ボタンによるタスク進行

その日の完了状態の表示

本リリースまでに作成したい機能

ルーティンの複数作成・切り替え機能

実行履歴の確認機能

タスク数のカスタマイズ機能

UIの改善（よりシンプルな操作性）

## 使用する技術スタック
使用するフレームワークは Ruby on Rails を想定しています。
データベースは PostgreSQL を使用します。
デプロイ先は Render を予定しています。

CRUD以外の実装としては、
・ルーティンの順番固定管理
・現在進行中のタスク状態管理
などを想定しています。
