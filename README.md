## サービス概要

本サービスは、
「今日誰に声をかければよいかを、考えずに判断できる状態」を作ることを目的としています。

ユーザーは当日の状態を
「遊べる・未定・無理」のいずれかで登録します。
この状態は前回の登録内容が保持されるため、
毎日必ず更新する必要はなく、変更がある場合のみ操作します。

操作はワンタップで完了し、
文章入力や細かな設定は必要ありません。

画面には友人全員のカードが一覧で表示され、
カードを見るだけで、その日に声をかけやすい相手を即座に把握できます。

既存のメッセージアプリでは、
「今遊べるかどうか」を確認するために
発言履歴を遡ったり、個別に声をかけたりする必要があります。

本サービスでは、
声をかける前の確認コストをなくすことで、
「迷っているうちに時間が過ぎてしまう」状況を防ぎ、
スムーズにゲームを始められる体験を提供します。

## このサービスへの思い・作りたい理由

オンラインゲームを遊ぶ際、「今日誰が遊べるのか」を確認するために、Discordや汎用カレンダー、メッセージアプリを行き来することに手間を感じていました。
特に、予定を立てるほどではないが「今日遊べる人がいれば遊びたい」というケースでは、既存のツールは過剰で使いづらいと感じていました。
そこで、予定管理ではなく「その日の参加可否」だけに絞った、ゲーム用途に最適化されたサービスを作りたいと考え、本サービスを企画しました。

## ユーザー層について

主なユーザー層は、オンラインゲームを日常的に遊ぶ友人グループです。
固定メンバーで遊ぶことが多く、当日の空き時間に合わせて突発的にプレイするケースが多いため、詳細な予定調整よりも「今日遊べるかどうか」を素早く共有できることに価値があると考え、この層を対象としました。

## サービスの利用イメージ

ユーザーは当日の状態を「遊べる・未定・無理」のいずれかで登録します。
画面には友人全員のカードが表示され、カードをめくることでその日の状態を確認できます。
これにより、「今日誰に声をかければよいか」を迷わず判断でき、スムーズにゲームを始めることができます。

## ユーザーの獲得について

まずは自身の友人グループ内で利用してもらうことを想定しています。
実際のゲームプレイ前後で使用してもらい、使いやすさや「遊べる人を把握しやすいか」といった点について、口頭やメッセージでフィードバックを集める予定です。
初期はクローズドな利用を前提とし、実体験をもとに改善を重ねていきます。

## サービスの差別化ポイント・推しポイント

本サービスは、Discord や汎用カレンダーのような既存ツールとは異なり、「今日遊べるかどうか」を瞬時に把握することに特化しています。

既存ツールでは、全員にメッセージを送って返信を待つ必要があったり、チャット履歴を遡って状況を確認する必要がありました。本サービスでは、友人全員の状態が一画面に集約されるため、確認の手間を最小限に抑えることができます。

また、友人の状態はカード形式で表示され、当日一度だけめくることができます。
この仕組みにより、単なるステータス確認ではなく、「今日は誰が遊べるのか」を確認する行為自体をゲーム的な体験として楽しめる点が特徴です。

ステータス更新という新しい行動を求める代わりに、
・確認の即時性
・操作の単純さ
・遊びに直結する体験
を提供することで、既存ツールでは代替しづらい価値を実現しています。

## 機能候補

MVPリリース時に実装予定の機能
・ユーザー登録／ログイン機能
・当日の状態（遊べる／未定／無理）の登録・更新・削除
・友人の状態をカード形式で一覧表示する機能
・カードを当日一度だけめくれる表示制御

本リリースまでに検討している機能
・友人グループ管理機能
・簡易的な通知機能
・UI/UXの改善

## 使用する技術スタック

アプリケーションは Rails を使用して作成する予定で、バージョンは安定して利用されているものを選びます。
データベースは PostgreSQL を利用し、予定・ユーザー・フォロー関係を保存します。
デプロイ先は Render または Rails で一般的に利用されるサービスを想定し、継続的に動かせる環境に置きます。
本アプリでは、ユーザー操作を起点に状態を取得する設計としているため、リアルタイム通信は使用しません。
MVP段階では実装のシンプルさと保守性を優先しています。
MVPではRailsのビューで一覧・スケジュール画面を実装し、本リリースではこれらの画面を中心にReactを導入することで、視覚的に分かりやすく、操作感の良いUIへの改善を予定しています。
